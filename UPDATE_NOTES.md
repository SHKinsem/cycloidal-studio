# 移距修型算法更新说明

## 更新日期
2025-10-07

## 主要修改

### 1. 修型算法变更
**原算法（全局周期）：**
```python
phi = (theta - theta_max + π) % (2π) - π
delta = offset + shift * (phi/π)²
```
- 在整个360°范围内形成单一的二次曲线修型
- 修型量在 theta_max 处最小，在 theta_max ± π 处最大

**新算法（逐齿周期）：**
```python
tooth_pitch = 2π / N  # 单齿角度宽度
phi = (theta % tooth_pitch) - tooth_pitch/2
delta = offset + shift * (phi / (tooth_pitch/2))²
```
- 在每个齿的范围内独立形成二次曲线修型
- 每个齿在齿中心处修型量最小，在齿间隙处最大
- 产生 N 个周期性修型（对应 N 个齿）

### 2. SolidWorks方程更新

**新的修型公式：**
```
delta(t) = DeltaOffset + DeltaShift * ((1 - cos(N*t))/2)
```

其中：
- `N = 14` (齿数)
- `(1 - cos(14*t))/2` 产生14个周期性调制，范围 [0, 1]
- 在每个齿中心 (t = k*2π/14, k=0,1,2,...13) 处，cos(14*t) = 1，调制量为0，修型量最小
- 在齿间隙处 (t = (2k+1)*π/14)，cos(14*t) = -1，调制量为1，修型量最大
- 修型量范围：[DeltaOffset, DeltaOffset + DeltaShift] = [-0.0388, -0.02] mm

**COMPACT 变体方程（推荐使用）：**

X(t):
```
34.0*cos(t)+2.0*cos(15*t) + ( -0.0388 + 0.0188*((1 - cos(14*t))/2) - 2.5)*( 34.0*cos(t)+2.0*15*cos(15*t) )/sqrt( (-34.0*sin(t)-2.0*15*sin(15*t))^2 + (34.0*cos(t)+2.0*15*cos(15*t))^2 )
```

Y(t):
```
34.0*sin(t)+2.0*sin(15*t) + ( -0.0388 + 0.0188*((1 - cos(14*t))/2) - 2.5)*( 34.0*sin(t)+2.0*15*sin(15*t) )/sqrt( (-34.0*sin(t)-2.0*15*sin(15*t))^2 + (34.0*cos(t)+2.0*15*cos(15*t))^2 )
```

### 3. 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| Rb | 34.0 mm | 针齿中心圆半径 |
| Rr | 2.5 mm | 针齿（滚子）半径 |
| E | 2.0 mm | 偏心距 |
| N | 14 | 摆线盘齿数 |
| DeltaOffset | -0.0388 mm | 等距修型基准量 |
| DeltaShift | 0.0188 mm | 移距修型振幅 |

### 4. SolidWorks 导入步骤

1. **创建全局变量**（在 Equations 中）：
   ```
   "Rb" = 34.0
   "Rr" = 2.5
   "E" = 2.0
   "N" = 14
   "DeltaOffset" = -0.0388
   "DeltaShift" = 0.0188
   ```

2. **插入方程驱动曲线**：
   - Insert > Curve > Equation Driven Curve
   - 选择 "Parametric" 类型
   - 参数范围：t = 0 到 6.28318 (2π)
   - 点数建议：500

3. **粘贴方程**：
   - 从 `solidworks_equations.txt` 复制 COMPACT 变体的 X(t) 和 Y(t)
   - 如果方程过长无法一次粘贴，可分两次：先粘贴 X(t) 并确认，再重新打开粘贴 Y(t)

4. **备选方案**（如果方程不工作）：
   - 使用点云导入：Insert > Curve > Curve Through XYZ Points
   - 导入 `cycloidal_modified.csv` 文件
   - 使用 Fit Spline 功能拟合曲线

## 技术优势

### 逐齿周期修型的优点：
1. **更精确的齿形控制**：每个齿独立修型，避免全局修型导致的不均匀性
2. **更好的啮合性能**：每个齿在啮合中心区域（齿中心）保持较小修型量，提高接触精度
3. **减少边缘效应**：齿间隙处增大修型量，减少应力集中
4. **周期一致性**：14个齿的修型模式完全一致，保证运动平稳性

### 与全局修型的对比：
| 特性 | 全局周期 | 逐齿周期 |
|------|---------|---------|
| 修型周期 | 1个/圈 | 14个/圈 |
| 齿形一致性 | 差 | 优 |
| 边界连续性 | 需要特殊处理 | 自然周期性 |
| 调整灵活性 | 低 | 高 |

## 2026-07-10 谐波修型 + GA 目标函数升级

### Bug 修复
- SolidWorks 导出相位反了半个齿距：`tooth_mod_formula` 由 `(1-cos(N*t))/2` 改为 `(1+cos(N*t))/2`，
  现在与 Python/CSV 几何同相位（此前 SW 方程的盘齿根最紧，15μm 公差叠加下会卡死）。

### 新参数化（推荐使用）
δ(θ) = offset + s1·cos(N·θ) + s2·cos(2N·θ)，沿轮廓法向，δ<0 去料。
- s1>0 齿顶紧 / s1<0 齿根紧 / s2<0 齿腰紧（经典"反弓"形状）
- 与 SW COMPACT 方程同构，导出零误差；旧 pertooth/sw 径向变体仅作历史对照。

### GA 目标函数
旧：最小化最大压力角。新：公差可行（margin≥0.5 arcmin @ 15μm 最坏叠加）前提下
最小化 背隙 + 0.05·波动 + 刚度不足惩罚。见 `CycloidalOptim_GA.py`。

### 本次最优解（Rb=34, Rr=2.5, E=1.5, N=18, T=30N·m, tol=15μm, seed=42, 80代）
offset=-0.0254, s1=-0.0034, s2=-0.0081 →
- 背隙 5.22 arcmin（当前 pertooth 设计 9.72）
- 刚度 33.9 N·m/arcmin（当前 29.1，反而更高：修型把最紧点放到齿腰，受载齿数 3→4）
- 波动 12.2 μrad（当前 104，丝滑度提升约 8 倍）
- 公差余量 0.68 arcmin（贴着 15μm 硬约束的边界；余量换齿数是本设计的核心权衡）

齿廓形状：齿腰(u≈0.25/0.75)最紧≈17μm→刚度；齿顶/齿根放松→吸收公差与入啮冲击。这就是"反弓"修型。

⚠️ 余量仅 0.68 arcmin，若实际公差叠加 > 15μm 会卡死。放宽 `TOL_BUDGET` 或提高 `MARGIN_MIN`
重跑即可得到更保守（余量更大、背隙略增）的解。背隙-余量为此消彼长，5.22 arcmin 已近约束下限。

### 交互式可视化
`cycloidal_viz.html`（单文件，离线可用）：啮合动画 + 齿廓/间隙曲线 + 三方案对比 + 参数游标。

### 文件变更
- 删除 `CycloidalOptim_GA,py`（废弃：全局修型公式 + Rb=65 旧参数），替代品 `CycloidalOptim_GA.py`

## 2026-07-10（二）对抗式评审修正 + 开箱即用设计器

经多智能体对抗式评审（物理/移植保真/GA目标/数值/HTML 五维，逐条独立复核），修正以下已确认缺陷：

### 物理指标修正（重要）
- **波动(丝滑度)指标口径错误**：原来取 `ptp(弹性变形 b−hi)`，漏掉了运动学消隙项 `hi`。真实负载
  传动误差是**绝对输出转角 `b=hi+弹性`** 的峰峰值。因 hi 与弹性近似反相，旧口径把波动高估约 2–4 倍，
  且误差符号随几何变化会翻转 → 会误排设计。已改为 `ripple = ptp(b)`（`windup` 返回绝对角 b）。
- **曲柄角采样过疏**：`PSI_SAMPLES` 由 6 点增至 48 点（一个啮合周期内），旧的 6 点把高频波动欠采约 44%。
  注：修正后的 `ptp(b)` 收敛较慢（48点约为收敛值的 65%），作为"量级/趋势"指标够用；追求精确值可再加密。
- **windup 承载不足分支**返回大有限哨兵值（不再返回 NaN），避免污染 `max/min` 与 GA 的 `argmax`。

### GA 目标函数修正（对齐"刚性最强"）
- 旧 cost 只对刚度设**下限惩罚** `max(0,25−stiff)`，从不奖励更高刚度 → GA 会为降背隙**牺牲刚度**。
  已改为**直接奖励刚度** `−0.08·stiff`（刚度<25 仍硬罚），真正对齐用户"刚性最强"目标。
- 可行性判据 `δ_max` 由保守上界 `offset+|s1|+|s2|` 改为**真实最大值**（二次型闭式解），不再误杀
  两谐波反相时更紧的可行齿廓。
- **多起点**（seed 42/7/2024）取全局最优，抗单种子局部最优。

### 本次最优解（Rb=34, Rr=2.5, E=1.5, N=18, T=30N·m, tol=15μm；取代上一节旧值）
**offset=-0.0209, s1=-0.0012, s2=-0.0037** →
- 背隙 **5.06 arcmin**（旧解 5.22；当前 pertooth 9.72）
- 刚度 **35.7 N·m/arcmin**（旧解 33.9；因新目标直接奖励刚度而更高）
- 波动 16.4 μrad（真实传动误差口径；旧解按新口径为 6.1，更平滑但刚度略低——smooth↔stiff 是权衡）
- 公差余量 0.69 arcmin，受载齿数 4，贴着 15μm 边界

⚠️ smooth 与 stiff 是**帕累托权衡**：新目标偏向刚度（用户强调"刚性最强"）。想更丝滑就把 s2 调更负
（齿腰更紧的反弓）——在 `cycloidal_viz.html` 里拖滑块即可实时看到权衡，无需重跑 GA。

### 其他修正
- `CycloidalOptim.py`：FULL_Y/SAFE_Y 的 SolidWorks 方程括号失衡（净 −2），已修并加**括号平衡断言**；
  第 7 节打印的修型公式仍是旧的 `(1−cos)/2`，已同步为 `(1+cos)/2` 及正确的齿顶/齿根描述。
- `cycloidal_viz.html`：live 指标点数 4000→8000（与 Python NTH 对齐，现逐位相符）；暂停后调参
  啮合面不再滞留旧图；尊重系统"减弱动效"（reduced-motion）默认暂停；公差余量"偏紧"增加非颜色提示。

### 开箱即用设计器（`cycloidal_viz.html`）
现在是完整的**摆线齿廓设计器**：可编辑几何与工况（Rb/Rr/E/N/公差/扭矩）→ 实时啮合+指标 →
**一键导出 SolidWorks 方程驱动曲线 X(t)/Y(t)（数值内联）+ 点云 CSV**；中英文切换。物理为
`CycloidalModAnalysis.py` 的 JS 移植，已在默认与非默认几何下与 Python 逐位核对。

## 2026-07-10（三）进阶层: 富参数化 + 多目标 + 对标论文

基于文献调研(ML 在此领域几乎只是"代理模型套经典 GA/NSGA-II";设计自由度的突破来自更丰富的
**几何参数化**而非神经网络), 把工具升级为可对标论文的多目标设计系统。核心不改验证过的物理核心,
以 `cycloidal_advanced.py` 叠加实现。

### 富参数化
δ(θ) = offset + Σ_{k=1}^{K} c_k·cos(kNθ) (余弦级数=双向对称+天然带限平滑), K=2 时精确退化为原 s1,s2。

### 新增目标(用户要求 + 文献)
- **压力角**: 受载接触点 力向量与节圆切向夹角(力加权)。实测修型使峰值 84.9°→55° (对标摆线理论)。
- **可加工性**: 齿廓凹处最小曲率半径, 须 ≥ 刀具半径(线切割~0.1mm)。
- **平滑度**: 曲率总变差(富参数化后防止齿廓抖动)。
- **鲁棒性**: ±机加工误差(TOL/2)下的最差公差余量。

### NSGA-II 多目标(`cycloidal_nsga2.py`, pygad, 无新依赖)
在 [offset, c1..c4] 上求 **帕累托前沿**: 背隙 / -刚度 / 波动 / 最大压力角, 硬约束 鲁棒余量≥0.5' +
凹曲率半径≥刀具 + 处处去料≥5µm。本次 1788 次评估 → **114 点前沿**(pareto_front.csv, ParetoFront.png)。
前沿揭示 **刚度↔压力角耦合**(推刚度必推高压力角)。膝点: offset -0.0483, c=[-0.0239,-0.0166,-0.0107,
-0.0035] → 背隙 8.90', 刚度 35.7, 波动 42.2µrad, 最大压力角 47.8°, 鲁棒余量 2.98', 4 齿。

### 对标验证(`cycloidal_benchmark.py`, 5/5 通过)
复现文献定性趋势: 修型降压力角[C]; 反弓优于纯等距[A]; 受载齿数随扭矩增加[D]; 背隙↔余量单调;
加工误差侵蚀余量[B]。**关键发现**: 上一节的名义3谐波最优(offset -0.0209)在 ±7.5µm 下 worst_margin
= -1.79' → **会卡死**; NSGA-II 的鲁棒前沿设计不会。这说明鲁棒约束是必需的。

### 交互浏览器(`cycloidal_pareto.html`)
自包含双语帕累托前沿浏览器: 内嵌 114 设计, 可选坐标轴/颜色的交互散点, 点击查看全部指标 + 一键复制
该设计的 K 谐波 SolidWorks 方程。主设计器(`cycloidal_viz.html`)已链接到它。
